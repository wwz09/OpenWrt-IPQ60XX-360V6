# 补丁变更说明

## 补丁名称：0001-fix-compilation-issues.patch

### 变更内容

1. **依赖库配置修复**
   - 在 `diy-part1-mediatek.sh` 中添加了防止重复添加仓库的检查逻辑
   - 确保 `openclash` 插件源只添加一次，避免重复条目导致的编译错误

2. **配置文件清理**
   - 在 `cmcc-rax3000m.config` 中移除了冲突包和不需要的插件：
     - 移除 `luci-app-cpufreq`
     - 移除 `luci-app-control-timewol`
     - 移除 `luci-app-oaf`
   - 添加了新的主题和应用市场插件：
     - 添加 `luci-app-design-config`
     - 添加 `luci-app-aurora-config`
     - 添加 `luci-app-store`

3. **网络配置修正**
   - 在 `diy-part2-mediatek.sh` 中修正了网络配置，将网络接口从 `eth0/eth1-4` 修改为 `wan/lan1-4`，以匹配 MediaTek MT7986A 平台的实际硬件
   - 确保网络配置与硬件接口名称一致，避免编译和运行时错误

4. **依赖处理优化**
   - 在 `diy-part2-mediatek.sh` 中添加了冲突包清理和 feeds 配置优化
   - 清理可能导致冲突的包，减少依赖缺失警告
   - 优化 feeds 配置，避免重复条目

5. **代码规范遵循**
   - 确保所有代码修改遵循项目的代码规范
   - 保持中文注释的完整性和清晰性
   - 确保脚本格式和风格与原项目一致

### 修复的问题

1. **依赖库重复添加问题**：避免了 `openclash` 插件源的重复添加
2. **配置文件冲突问题**：移除了可能导致冲突的插件
3. **网络配置不匹配问题**：修正了网络接口配置，与硬件实际接口一致
4. **依赖缺失警告问题**：优化了依赖处理，减少了依赖缺失警告
5. **代码规范问题**：确保所有修改遵循项目代码规范
